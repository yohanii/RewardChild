## 🚀 전체 개발 순서 (MVP 기준)

### 1️⃣ **아이디어 명확화 & 기능 정리 (기획 단계)**

- **핵심 목표 정의**: 앱이 해결하고 싶은 문제 1개를 명확히.
- **핵심 사용자 시나리오**: 사용자가 앱에서 처음부터 끝까지 어떤 행동을 하는지 간단히 정리 (예: 회원가입 → 습관 설정 → 진행 체크 → 보상 지급).
- **MVP 기능만 추림**: 꼭 필요한 기능만 리스트업.
    
    예:
    
    - 회원가입 / 로그인
    - 습관 등록 / 체크
    - 파트너 연결
    - 포인트 or 리워드 표시

📄 *결과물*: Notion, FigJam, Miro 등으로 기능 플로우 정리 (기획서 수준이면 충분)

---

### 2️⃣ **UX/UI 설계 (디자인 단계)**

- **핵심 플로우 기준으로 와이어프레임 제작**
    
    → Figma에서 빠르게 화면 흐름 설계
    
- MVP는 “예쁘게”보다 **직관적으로 작동하는 게 중요**
- RN으로 구현하기 쉬운 컴포넌트 위주로 (BottomTab, Modal, Button 등)

📄 *결과물*: Figma 시안 (핵심 화면: 로그인 / 홈 / 상세 / 설정)

---

### 3️⃣ **데이터 모델링 & 백엔드 설계**

- Supabase에서 **테이블 설계**
    - 예:
        - `users` (id, email, name, balance, created_at)
        - `habits` (id, user_id, title, reward, status, created_at)
        - `partners` (id, user_id, partner_id, status)
    - 외래키 관계 설계하고 Supabase에서 RLS(Row Level Security) 설정
- **인증 로직(Supabase Auth)** 먼저 테스트
    
    → 이메일 로그인, OAuth 필요 여부 결정
    

📄 *결과물*: Supabase 프로젝트 + SQL schema 문서

---

### 4️⃣ **React Native 환경 세팅**

- Expo CLI로 빠르게 세팅
    
    ```bash
    npx create-expo-app my-habit-app
    
    ```
    
- Supabase JS SDK 설치
    
    ```bash
    npm install @supabase/supabase-js
    
    ```
    
- `.env`에 Supabase URL, anon key 세팅

📄 *결과물*: RN 초기 세팅 완료, Supabase 연결 테스트

---

### 5️⃣ **기능 단위 개발 (Frontend + Backend 연동)**

개발은 화면 단위보다 **기능 단위로 나누는 게 효율적**:

1. Auth (회원가입, 로그인)
2. 습관 CRUD
3. 리워드 지급/확인
4. 파트너 초대
5. 알림 or 통계 (나중 단계)

→ 각 기능 단위로 “UI → API 연동 → 테스트” 순서로 구현

---

### 6️⃣ **테스트 & 디버깅**
